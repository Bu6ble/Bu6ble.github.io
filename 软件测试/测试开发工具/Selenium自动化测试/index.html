自动化测试理论

  • UI：User Interface（用户接口-用户界面），主要包括：web、app
  • UI自动化测试：使用工具或代码执行用例的过程
  • 什么样的项目适合做自动化
    • 需要回归测试的项目（甲方自营项目、金融、电商）
    • 需求变动不频繁：稳定的模块
    • 项目周期长的项目（甲方自营项目、6个月以上的外包）

Selenium实现元素定位及操作

元素定位

  • 什么是元素定位

    • 通过代码调用方法查找元素

  • 元素定位方法

    定位方式 类型 其他说明
    id 利用元素属性 有ID优先使用ID
    name 利用元素属性 无ID看name
    class_name 利用元素属性 无name看class
    link_text 超链接定位 全部文本信息
    partial_link_text 超链接定位 局部文本信息
    Xpath 路径定位 路径（相对路径）
    属性
    属性与逻辑结合
    属性包含
    利用元素文本
    CSS 选择器定位 ID选择器
    class选择器

    属性选择器
    元素选择器
    层级选择器
    属性包含

  • 结论：

    • 首推CSS定位，原因：执行速度快

      • 如果有ID属性，使用#id
      • 没有id属性，使用其他有的属性（能代表唯一的属性）
      • 如果属性都不能代表唯一，使用层级

    • 如果CSS解决不了，则用XPath

  • 步骤

    • 打开谷歌浏览器
    • 输入url
    • 查找元素及操作
    • 关闭浏览器
  • 在 Selenium 4 及之后的版本中，原本的 find_element_by_* 系列方法（像 find_element_by_id、find_element_by_xpath 等）已经被弃用，取而代之的是统一的 find_element() 方法结合 By 类来定位元素。
  • 需要把 find_element_by_id 替换成 find_element() 方法，并结合 By.ID 来定位元素。

  • id定位

    • 方法：driver.find_element(By.ID, "id值")
      • Selenium 4 之前是driver.find_element_by_id("id值")
    • 前置：标签必须有id属性
    • 输入方法：元素.send_key("内容")

  • name定位

    • 方法：driver.find_element(By.NAME, 'your_element_name')
      • Selenium 4 之前是driver.find_element_by_name("name属性值")
    • 前置：标签必须含有name属性
    • 特点：当前页面可以重复
    • 提示：由于name属性值可以重复，所以使用时需要查看是否为唯一

  • class定位

    • 方法：driver.find_element(By.CLASS_NAME, 'your_class_name')
      • Selenium 4 之前是driver.find_element_by_class_name("class属性值")
    • 前置：标签必须有class属性
    • 特点：class属性值可以有多个值
    • 提示：如果标签有多个class值，使用任何一个都可以

  • tag_name定位

    • 方法：driver.find_element(By.TAG_NAME, 'your_tag_name')
      • Selenium 4 之前是driver.find_element_by_tag_name('your_tag_name')
    • 说明：根据标签名进行定位
    • 提示：如果页面存在多个相同标签，默认返回第一个

  • link_text定位

    • 方法：driver.find_element(By.LINK_TEXT, 'your_link_text')
      • Selenium 4 之前是driver.find_element_by_link_text('链接文本')
    • 说明：根据链接文本（a标签）进行定位
    • 特点：传入的链接文本，必须全部匹配，不能模糊

  • partial_link_text定位

    • 方法：driver.find_element(By.PARTIAL_LINK_TEXT, 'partial_link_text')
      • Selenium 4 之前是driver.find_element_by_partial_link_text('链接文本')
    • 说明：根据链接文本（a标签）进行定位
    • 特点：传入的链接文本，支持模糊匹配（传入局部文字）

  • 【扩展】查找一类元素

    • 方法：driver.find_elements(By.ID, 'your_element_id')
      • Selenium 4 之前是driver.find_elements_by_xxx()
    • 说明：返回列表格式，要使用需要添加下标或遍历
    • 提示：八大元素定位方法，都可以使用一组元素定位，如果没有搜索道符合的标签，返回空列表
  • 如果标签没有（id\name\class）3个属性，也不是链接标签，只能用tag_name定位
  • 方便在工作中查找元素，使用xpath和css比较方便（支持任意属性、层级）来找元素

xpath定位

  • xpath是xml path简称，使用标签路径来定位
  • 绝对路径：从根目录开始，逐级查找标签

  • 相对路径：从任意层级开始，查找标签

  • 策略（方法）

    • 路径
    • 属性
    • 属性与逻辑（多个属性）
    • 属性与层级（路径）

  • 方法：element = driver.find_element(By.XPATH, 'your_xpath_expression')

    • 在Selenium 4之前的方法driver.find_element_by_xpath(“表达式”)

  • 属性：

    • 单属性：//*[@属性名=’属性值‘]
    • 多属性：//#[@属性名='属性值' and @属性名='属性值']
    • 提示：可以使用任何属性

  • 层级与属性：

    • 如果元素现有的属性不能唯一匹配，需要结合层级使用
    • 语法
      • // 父标签/子标签 必须是直属子级
      • // 父标签[@属性名=’属性值’] // 后代标签 父和后代之间可以跨越元素
    • 扩展
      • 根据显示文本定位：//*[text()='文本值']
      • 属性值模糊匹配：//*[contain(@属性名, '属性部分值')]

CSS选择器定位

  • 执行速度快：
    • 浏览器原生支持：现代浏览器对 CSS 选择器有专门的解析和查询引擎。因为 CSS 在网页样式渲染中被广泛使用，浏览器在解析网页时，会对 CSS 选择器进行优化，Selenium 利用了浏览器的这一特性，使得通过css_selector定位元素时能够快速得到结果。
    • 简洁高效的语法：CSS 选择器语法相对简洁，能够以非常直接的方式定位元素。例如，可以通过标签名、类名、ID 等直接定位，像div#my - id能快速定位到 ID 为my - id的div元素。相比于 XPath 复杂的语法，CSS 选择器在构建和解析时都更高效，减少了查找元素时的计算量。
    • 局部定位能力：它可以基于元素的局部特征进行定位，不需要像 XPath 那样从文档根节点开始遍历整个 DOM 树。比如，当要定位某个特定父元素下的子元素时，css_selector能利用父子关系快速筛选，如parent - class > child - class，这种局部性查找大大缩小了查找范围，提高了查找速度。
  • 策略：
    • id选择器：#id属性值
    • 类选择器：.class属性值
    • 标签选择器：标签名 ——标签要看是否在页面中唯一，否则返回单个或所有
    • 属性选择器：[属性名 = '属性值'] ——标签中的任何一个属性
      • 多个的话：[属性名1 = '属性值1'] [属性名2 = '属性值2']
    • 层级选择器：
      • 父子关系：选择器 > 选择器 如：#p1 > input
      • 后代关系：选择器 选择器 如：#p2 input

        • 如果说跳转的是在当前窗口跳转则会聚焦到新网页中

Selenium实现等待

  • 什么是等待？

    • 代码执行过程中，第一次未找到元素，先不抛出异常。激活等待时间，在等待过程中如果找到元素就执行。

  • 为什么要等待？

    • 由于网络或配置原因，导致元素未加载出来，而代码已执行，会触发异常。

  • 元素的等待类型

    • 隐式等待
    • 显式等待
    • 强制等待 sleep()
  • 隐式等待 显式等待
    作用域 只用设置一次，对全局生效 只对指定元素生效
    抛出异常 NoSuchElementException TimeoutException
    使用方法 driver.implicitly_wait(timeout) 通过WebDriverWait对象
    其他 必须等待整个页面加载完成 只关注指定元素是否加载

隐式等待

  • 针对全部元素生效
  • 方法：driver.implicitly_wait(秒)
    • 定位元素时，如果能定位到元素则直接返回该元素，不触发等待
    • 如果不能定位到元素，则间隔一段时间后再去定位元素
    • 如果在达到最长时长时还没有找到指定元素，则抛出元素不存在的异常NoSuchElementException
  • 提示：在项目中，未封装自动化框架时，推荐使用

显示等待

  • 针对单个元素生效，可以修改查找频率和超时时间
    • 定位指定元素，如果能定位到元素则直接返回该元素，不触发等待
    • 如果不能定位到元素，则间隔一段时间后再去定位元素
    • 如果在达到最长时长时还没有找到指定元素，则抛出超时异常TimeoutException
  • 特点：查找并返回元素
  • 实现方法：
    • 导包：from selenium.webdriver.support.wait import WebDriverWait
    • 创建显示等待类对象：WebDriver(driver, timeout, poll_frequency=0.5)
      • poll_frequency ：访问频率
    • 调用utils方法：until(method)：直到...时

强制等待

  • 语法：sleep(10)
  • 提示：执行到这句必须等待10秒，不灵活

Selenium实现Frame框架元素操作

  • frame（iframe）标签作用：在页面中加载另一个页面
  • 为什么处理frame：焦点默认在启动页面，如果不处理iframe，无法操作iframe嵌入的页面元素
  • 如何处理：
    • 步骤：
      • 切换到iframe：driver.switch_to.frame(iframe元素)
      • 操作元素
      • 回到默认页面：driver.swtich_to.default_content()
      • 扩展：
        • **driver.switch_to.parent_frame()**：适用于需要在嵌套的iframe结构中逐级向上切换焦点的情况，每次调用只会向上移动一层。
        • **driver.switch_to.default_content()**：适用于需要快速跳出所有嵌套的iframe，直接回到页面最顶层的情况，无论当前嵌套的层级有多深。

Selenium实现验证码处理

  • 验证码：

    • 去掉验证码、万能验证码
    • 验证码识别技术
    • 记录cookie

  • 关于cookie

    • cookie由Web服务器生成，并且保存在用户浏览器上的小文本文件，可以包含用户相关的信息
    • cookie的数据格式：键值对组成（python中的字典）
    • cookie的产生：客户其请求服务器，如果服务器需要记录该用户状态，就向客户端浏览器颁发一个cookie数据
    • cookie的使用：当浏览器再次请求该网站时，浏览器把请求的数据和cookie数据一同提交给服务器，服务器检查该cookie，以此来辨认用户的状态

  • 如何使用cookie跳过登录

    • 手工登录
    • 获取cookie
    • 脚本添加cookie
    • 刷新页面

  • cookie操作方法

    • 获取指定cookie：driver.get_cookie(name)
    • 获取本网站所有本地cookies：driver.get_cookies()
    • 添加cookie：driver.add_cookie(cookie_dict)
      • cookie_dict：一个字典对象，必选的键包括name和value

PO模式

  • 存在的问题分析：
    • 元素定位的代码对页面特别依赖，一旦页面发生变化所有用到该页面的元素定位代码都需要进行修改
    • 存在大量元素定位代码冗余
  • PO——Page Object的缩写，PO模式是自动化测试项目开发实践的最佳设计模式之一
    • 将自动化涉及的页面或模块封装成对象
    • 解决的问题：
      • 代码复用性
      • 便于维护（脚本层与业务分离）
        • 如果元素信息发生变化时，也不要去修改脚本
    • 页面：每个页面都是对象
    • 页面上的元素：定义每个元素对象定位信息的实例属性
    • 页面上的操作：基于业务功能定义页面所提供的业务方法
  • PO如何做？
    • Base层：存放所有Page页面公共方法
    • Page层：基于页面或模块单独封装当前页面要操作的对象
    • Script层：脚本层+unittest
