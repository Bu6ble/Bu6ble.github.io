移动自动化测试

  • 主流的移动自动化测试工具

    Robotium macaca Appium
    介绍 是一款国外开源的Android自动化测试框架 是由阿里巴巴公司开源的一套自动化解决方案 是一款国外开源的自动化测试工具
    适用平台 Android PC端、Android、iOS Android、iOS
    支持语言 Java Java、Python、Node.js Java、JS、PHP、Python、C#、Ruby等主流语言
    应用场景 不支持跨应用 支持跨应用 支持跨应用

  • 安装流程

    • 安装jdk1.8

      • 安卓应用或开发工具使用java语言开发，必须使用jdk
      • 配置环境变量：JAVA_HOME='C:\XXX\jdk1.8.0_151'（这里为JDK安装目录）
        • 在Path中添加：%JAVA_HOME%\bin

    • 安装Android SDK：https://developer.android.google.cn/tools/releases/platform-tools?hl=zh-cn ；https://www.androiddevtools.cn/

      • 安装tool和platform-tools，主要使用adb命令工具和uiautomatorviewer工具

      • 环境变量里加解压后的路径

        • ADNROID_HOME=D:\Android\sdk
        • 在path中添加：%ANDROID_HOME%\tools;%ANDROID_HOME%\platform-tools

      • 输入adb --version确认是否正常执行

    • 安装Appium：https://github.com/appium/appium-desktop/releases

      • 将脚本发送给手机

    • 安装模拟器

    • 安装appium-python库：pip install Appium-Python-Client

      • 自动化测试使用api

Appium的特点

  • 开源
  • 支持Android、iOS
  • 支持Native App、Web App、Hybrid App
    • Native App：原生应用，使用Android或者iOS的标准SDK编写的应用
    • Web App：移动浏览器应用，使用移动平台的浏览器访问的应用
    • Hybrid App：混合应用，把一个基于webview实现的功能进行包装的应用
  • Server也是跨平台的，可以使用Mac OS X、Windows或Linux
  • 用Appium自动化测试不需要重新编译App
  • 支持多语言编写脚本

Appium的设计原理

  • C/S架构，Appium的核心是一个web服务器，提供了一套接口。它会接收客户端发送过来的命令，然后在移动设备上运行命令，最后把运行结果通过HTTP响应包返回给客户端。
  • session，每个client连接到server以后都会创建一个session，自动化始终围绕一个session进行。

ADB工作原理

  • ADB = Android Debug Bridge，是一个手机调试工具

  • Client端：运行在开发机器中，即你的开发电脑，用来发送ADB命令
  • Server端：运行在开发机器中，用来管理Client端和手机的Daemon之间的通信
  • Daemon守护进程：运行在调试设备中，手机或模拟器，用来接收并执行ADB命令

ADB常用命令

获取设备列表

  • 输入下述命令获取设备列表：adb devices

  • 对于mumu模拟器，需要先在模拟器shell目录下启动：adb connect 127.0.0.1:xxxx（xxxx为ADB端口号）

    • 参考链接：https://mumu.163.com/help/20230214/35047_1073151.html

获取包名和界面名

  • 包名（package）：决定程序的唯一性（不是应用的名字）
  • 界面名/启动名（activity）：一个界面名，对应着一个界面
  • 命令
    • Mac/Linux：adb shell dumpsys window | grep usedApp
    • Windows：adb shell dumpsys window | findstr usedApp
  • 在获取之前必须先手工进入到索要获取的信息界面
    • 进入设置页面，返回信息为如下所示：
      • com.android.settings为包名
      • .Settings为启动名

文件传输

  • 上传文件：adb push 电脑文件路径 手机的文件路径
  • 下载文件：adb pull 手机的文件路径 电脑文件路径

启动时间

  • 启动app：adb shell am start -W 包名/启动名
    • ThisTime：该界面（activity）启动耗时（毫秒）
    • TotalTime：应用自身启动耗时 = ThisTime + 应用 application等资源启动时间（毫秒）
    • WaitTime：系统启动应用耗时 = Totaltime + 系统资源启动时间（毫秒）
  • 冷启动：应用已关闭，使用命令启动
  • 热启动：应用已打开在后台或当前页面，使用命令看启动时间
  • 测试通过的标准
    • 需求有明确的启动时间指标
    • 参考同类文件，启动时间不能大于竞争对手的启动时间

获取手机日志

  • 获取日志命令：adb logcat
  • 保存文件：adb logcat > 电脑文件路径\xxx.log
  • 对App进行操作时，需要开启日志，记录App操作的步骤和异常
  • 获取错误日志步骤
    • 当测试过程中发现问题后想获取错误日志信息
      • 打开被测应用程序，进入到触发缺陷的位置
      • 使用查看日志命令：adb logcat
      • 触发缺陷
      • 获取日志信息
  • 注意：一般留意Error错误级别的日志，以及类似下图的一些异常错误日志信息。触发错误日志时及时ctrl+c掐断日志刷新。

其他常用命令

命令 说明
adb install 路径/xx.apk 安装app到手机上
adb uninstall 包名 卸载手机上的app，需要指定包名
adb devices 获取当前电脑已经连接的设备和对应的设备号
adb shell 进入到安卓手机内部的linux系统命令行中
adb start-server 启动adb服务端，出bug时使用可以重启服务器，先关闭再启动
adb kill-server 停止adb服务端，出bug时使用可以重启服务器，先关闭再启动
adb –help 查看adb帮助
adb connect ip:端口 连接手机/模拟器
  • adb start-server 一般不需要人工启动，自动启动 adb.exe进程，当应用进程死机，需要执行杀服务，杀完需要执行命令启动。

UIAutomatorViewer查看元素信息

  • 目前在Android 8.0以下不能使用

基础API

代码启动说明

  • 启动应用
    • 方法：driver.start_activity(包名，启动名)
    • 说明：appium支持跨应用，可以操作应用中切换到B应用
  • 获取当前包名、启动名
    • 当前应用包名：driver.current_package
    • 当前应用启动名：driver.current_activity
  • 关闭APP：driver.close_app()
  • 关闭驱动：driver.quit()
  • 安装APP：driver.install_app(app_path)
  • 卸载APP：driver.remove_app(app_id)
    • app_id就是APP的包名
  • 判断APP是否安装：driver.is_app_installed(app_id)
  • 置于后台：background_app(seconds)

元素的定位

定位单个元素

  • 通过UIAutoMatorViewer获取到元素信息

  • ID定位：driver.find_element_by_id(resource_id属性值)

  • class定位：driver.find_element_by_class_name(class属性值)

  • xpath定位：driver.find_element_by_xpath(xpath表达式)

  • name定位：driver.find_element_by_acessibility_id(content-desc属性值)

  • 当定位到多个符合条件的元素时，默认返回第一个

定位一组元素

  • 通过UIAutoMatorViewer获取到元素信息
  • ID定位：driver.find_elements_by_id(resource_id属性值)
  • class定位：driver.find_elements_by_class_name(class属性值)
  • xpath定位：driver.find_elements_by_xpath(xpath表达式)
  • name定位：driver.find_elements_by_acessibility_id(content-desc属性值)
  • 定位一组元素，返回的是列表

元素操作

点击、输入和清空

  • 模拟操作
    • 点击：element.click()
    • 输入：element.send_keys(value)
    • 清空：element.clear()
  • 默认输入中文无效，但不会报错，需要在“前置代码”中加入两个参数
    • desired_caps['unicodeKeyboard'] = True
    • desired_caps['resetKeyboard'] = True

文本、位置和大小

  • 获取文本：element.text
  • 获取位置：element.location
  • 获取大小：element.size

获取元素属性值

  • 获取属性值：element.get_attribute(属性名)
  • 一些获取属性值对应的属性名和实际UIAutoMatorViewer显式不一定一致
    • 获取 resource-id：resourceId
    • 获取content-desc：name
    • 获取class：className
    • 获取text：text

滑动

  • swipe(start_x, start_y, end_x, end_y, duration)
    • 从一个坐标位置滑动到另一个坐标位置，只能是两个点之间的滑动
    • 参数说明：
      • start_x, start_y：起点坐标
      • end_x, end_y：终点坐标
      • duration：滑动操作持续的时间长度，单位：ms
    • 特点：精准滑动（基于两个坐标 点滑动），适合多次滑动，duration时间越长惯性越大
    • 说明：针对坐标点进行操作，实现自适应则需要通过获取滑动区域元素位置和大小来动态计算。
    • driver.swipe(100, 1000, 100, 400, duration=2000)
  • scroll
    • 从一个元素滑动到另一个元素，直到页面自动停止
    • 特点：滚动（有惯性存在，滚动下不按下第一个元素）
    • 说明：针对两个元素进行操作，只适合滑动一次的操作
    • driver.scroll(origin_el, destination_el)
      • origin_el：滑动开始的元素
      • destination_el：滑动结束的元素
  • drag_and_drop
    • 特点：拖拽（没有惯性，按下开始元素拖拽到指定元素位置）
    • 说明：针对两个元素进行精准操作
    • driver.drag_and_drop(origin_el, destination_el)
      • origin_el：滑动开始的元素
      • destination_el：滑动结束的元素

手势操作TouchAction

  • 实现步骤
    • 创建TouchAction对象
      • touch_action = TouchAction(driver)
    • 调用高级手势对象提供所想执行的手势方法
      • touch_action.手势方法
    • 执行手势
      • touch_action.perform()
  • 所有手势都要通过执行perform()函数才会运行
  • 直接连续一行进行编写：TouchAction(driver).手势方法.perform()
  • appium中封装了手势的操作方法，都在TouchAction类中，需要导包
    • from appium.webdriver.common.touch_action import TouchAction

TouchAction—轻敲

  • 模拟手指对某个元素或坐标按下并快速抬起
  • 实现方法
    • 一行实现：TouchAction(driver).tap(el, x, y).perform()
    • 创建TouchAction对象
      • touch_action= TouchAction(driver)
    • 调用轻敲，元素对象或坐标二选一
      • toucha_action.tap(element=None,x=None,y=None)
    • 执行手势
      • touch_action.perform()

TouchAction—按下与释放

  • 模拟手指一直按下，模拟手指抬起，可以组合成轻敲或长按的操作。
  • 实现方法：
    • 一行实现按下：TouchAction(driver).press(el,x,y).perform()
    • 一行实现抬起：TouchAction(driver).release().perform()
    • 创建TouchAction对象
      • touch_action = TouchAction(driver)
    • 调用按下，元素对象或坐标二选一
      • touch_action.press(el=None, x=None, y=None)
    • 调用抬起
      • touch_action.release()
    • 执行手势
      • touch_action.perform()

TouchAction—长按

  • 模拟手指对元素或坐标的长按操作，有些场景需要长按才能激活菜单选项
  • 实现方法：
    • TouchAction(driver).long_press(el=None, duration=1000).perform()
    • TouchAction(driver).long_press(x=None, y=None, duration=1000).perform()

TouchAction—移动和思考时间

  • 手指移动：模拟手指移动操作
  • 思考时间：模拟手指等待，再执行下一步时等待指定时间
  • 实现方法：
    • TouchAction(driver).move_to(el=None, x=None, y=None).perform()
    • TouchAction(driver).wait(ms=time).perform()

手机操作

获取分辨率

  • 通过获取手机分辨率来动态计算滑动的起止坐标点
  • 获取手机分辨率：driver.get_window_size()

获取手机截图

  • 有些自动化操作可能没有反应，但并不报错，可以将操作过后的关键情况，截图留存。
  • 实现方法：driver.get_screenshot_as_file(file_name)

获取手机网络

  • driver.network_connection

设置手机网络

  • driver.set_network_connection(connectionType)
  • connectionType：表示网络设置类型

操作通知栏

  • 打开通知栏实现方法：driver.open_notifications()
  • 没有提供关闭通知的api，那么按照现实操作即可，如手指从下往上滑，或者按返回键

其他API

获取Toast

  • toast消息为移动应用中，一种黑底白字提示信息，有事件限制
  • 提示信息不能直接定位，需要测试提示信息是否正确（断言内容）
  • 如何获取
    • 安装实现：pip install uiautomator2
    • 在配置信息里启用：desired_caps['automationName'] = 'Uiautomator2'
    • 编写代码获取文本值

webView APP测试

  • APP中嵌套WEB信息，如果不切换无法定位
  • 如何切换
    • 获取当前环境：driver.contexts
    • 调用切换方法：driver.switch_to.context("环境")

Monkey

  • 主要用于Android的压力测试，自动化的压测小工具，为了测试APP是否会crash（闪退/瘫痪/崩溃）
  • 安装：不需要安装，由Android系统自带，使用Java语言写成，在Android文件系统的存放路径是：/system/framework/monkey.jar
  • 启动方式：
    • cmd执行：adb shell monkey (+命令参数)进行Monkey测试
    • adb shell执行：monkey (+命令参数)
      • 指定app包：-p (被测试的app包名)
        • 用此参数指定一个或多个包，指定包之后，monkey将只允许系统启动指定的app。如果指定包列表，monkey将允许系统启动设备中的所有app。
        • adb shell monkey -p fishjoy.control.menu -p cn.goapk.market 100
          • 100为执行次数
      • 日志级别：-v
        • 通过多个-v的个数来指定查看日志级别
        • -v：缺省值，仅提供启动提示，测试完成和最终结果等少量信息
        • -v -v：提供较为详细的日志，包括每个发送到Activity的事件信息
        • -v -v -v：最详细的日志，包括了测试中选中/未选中的Activity信息
        • adb shell monkey -p cn.goapk.market -v -v 100
      • 随机种子数
        • （随机数种子）用于指定伪随机数生成器的seed值，如果seed相同， 则两次Monkey测试所产生的事件序列也相同
        • 看bug能不能复现
        • monkey测试1：adb shell monkey -p cn.goapk.market -s 10 100
        • monkey测试2：adb shell monkey -p cn.goapk.market -s 10 100
      • 事件间隔时间：
        • –throttle <毫秒>
        • adb shell monkey -p cn.goapk.market --throttle 3000 100
  • 常用执行命令：adb shell monkey -p com.yunmail.lc -v -v 10000 > xxx.log
    • -p：包名
    • -v -v 日志
    • 10000：乱抓的时间次数
    • -s：设置随机种子数（两次执行随机种子数一样，执行的事件也一样）
  • 日志查看什么
    • 搜索：ANR、ERROR、null、相关错误

adb shell monkey -p com.yunmail.lc -v -v --pct-pinchzoom 100 10000 > xxx.log

  • 执行缩放命令100%
  • 如果不设置则会随机进行上述11个事件
