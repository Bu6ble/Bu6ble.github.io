如何开发pytest插件

1. pytest的插件系统

  • 插件系统：pluggy
  • 基本原理
    • 【约定】声明一个函数，称之为hook
    • 【软件】软件运行过程中，自动调用hook
    • 【插件】插件会按照hook实现函数，被软件自动调用

2. pytest中的hook清单

  • 方式一：查看源码 hookspec.py

  • 方式二：查看文档 https://docs.pytest.org/en/latest/reference/reference.html#pytest.PytestPluginManager.hook

  • 插件如果提供了新的hook，需要通过代码的方式动态获取

3. hook规则

使用pdm管理项目

  • 如果是一个项目，最好是直接new project，而不是在现有的文件夹菜单栏进行创建

  • 步骤：

    • 选择项目位置
    • 创建venv
    • 安装pdm（项目管理工具）
    • 使用pdm初始化项目
    • 其他全部默认，回车即可

  • 在pyproject.toml中指定插件的入口点和源码目录

需求分析

1. 梳理需求

  • 主要需求：当pytest执行结束之后，自动将结果发送到钉钉、微信、飞书、Email等处
    • “结果”包括什么内容
      • 测试开始时间
      • 测试结束时间
      • 测试执行时间
      • 测试用例总数量
      • 测试用例成功的数量
      • 测试用例失败的数量
      • 测试用例通过率
      • 测试报告地址
    • 发送到“哪里”
      • 钉钉：个人、群
      • 微信：个人、群、公众号
      • 飞书：个人、群
    • 怎么发送
      • 个人：加好友、发送好友消息
      • 群：群机器人，调用接口
      • 公众号：关注公众，调用接口
      • Email：SMTP服务器，发送
    • “什么时候”发送
      • 每次测试结束
      • 有用例失败

2. 编码思路

伪代码

  • hook
    • 测试开始时自动执行，记录开始时间
    • 测试结束时自动执行，记录结束时间，计算执行时长，发送结果
    • 收集完测试用例之后自动执行，记录总的用例数量
    • 每个用例执行结束之后自动执行，记录当前用的结果
  • API
    • 微信：添加机器人后，有接口和接口说明

3. 测试用例

  • 单元测试
    • 测试开始时间、执行时长是否准确
    • 用例执行结果，是否准确
    • 用例数量，是否准确
    • 用例通过率，是否准确
    • API或SMTP，是否可用
  • 集成测试
    • 测试结束时，在微信群中，收到结果通知
