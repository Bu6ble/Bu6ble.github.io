有序数组转换成二叉搜索树

  • 【108. 将有序数组转换为二叉搜索树】：https://leetcode.cn/problems/convert-sorted-array-to-binary-search-tree/description/?envType=study-plan-v2&envId=top-100-liked
  • 题目：给你一个整数数组 nums ，其中元素已经按 升序 排列，请你将其转换为一棵 平衡 二叉搜索树。
  • 思路：
    • nums=[−10,−3,0,5,9]，我们从数组正中间的数 nums[2]=0 开始，把数组一分为二，得到两个小数组：
      • 左：[−10,−3]。
      • 右：[5,9]。
    • 答案由三部分组成：
      • 根节点：节点值为 nums[2]=0。
      • 把 nums[2] 左边的 [−10,−3] 转换成一棵平衡二叉搜索树，作为答案的左儿子。这是一个和原问题相似的子问题，可以递归解决。
      • 把 nums[2] 右边的 [5,9] 转换成一棵平衡二叉搜索树，作为答案的右儿子。这是一个和原问题相似的子问题，可以递归解决。
    • 递归边界：如果数组长度等于 0，返回空节点。

  • 复杂度分析：

    • 时间复杂度：O(n)，其中 n 是 nums 的长度。每次递归要么返回空节点，要么把 nums 的一个数转成一个节点，所以递归次数是 O(n) 的，所以时间复杂度是 O(n)。需要注意，Python 的第一种写法有切片的复制开销，二叉树的每一层都需要花费 O(n) 的时间，一共有 O(logn) 层，所以时间复杂度是 O(nlogn)；第二种写法避免了切片的复制开销，时间复杂度是 O(n)。

    • 空间复杂度：O(n)。如果不计入返回值和切片的空间，那么空间复杂度为 O(logn)，即递归栈的开销。

二叉树的最大深度

  • 【104：二叉树的最大深度】：https://leetcode.cn/problems/maximum-depth-of-binary-tree/description/?envType=study-plan-v2&envId=top-100-liked
  • 题目：
    • 给定一个二叉树 root ，返回其最大深度。
    • 二叉树的 最大深度 是指从根节点到最远叶子节点的最长路径上的节点数。
  • 思路：
    • 递归的思路
      • 递就是“从上到下，不断向下层下探的过程”
      • 归就是“从下到上，往上层回溯的过程”
      • 一开始，在还没能通过到 递归终结条件 获取边界值，那就不断下探寻找。一旦找到了这个所谓的边界值，那就根据这个边界值的答案，不断向上层带入，从而进一步获取上一层的答案。
    • 计算机如何执行递归
      • 当程序执行“递”动作时，计算机使用栈保存这个发出“递”动作的对象，程序不断“递”，计算机不断压栈，直到边界时，程序发生“归”动作，正好将执行的答案“归”给栈顶元素，随后程序不断“归”，计算机不断出栈，直到返回原问题的答案，栈空
  • 解法一：自底向上
  • 解法二：自顶向下
    • 维护全局变量，使用二叉树遍历函数，不断更新全局变量最大值
    • 重新赋值（替换）用nonlocal

  • 复杂度分析

    • 时间复杂度：O(n)，其中 n 为二叉树的节点个数。

    • 空间复杂度：O(n)。最坏情况下，二叉树退化成一条链，递归需要 O(n) 的栈空间。

相同的树

  • 【100.相同的树】：https://leetcode.cn/problems/same-tree/
  • 题目：
    • 给你两棵二叉树的根节点 p 和 q ，编写一个函数来检验这两棵树是否相同。
    • 如果两个树在结构上相同，并且节点具有相同的值，则认为它们是相同的。
  • 思路：
    • 边界条件：如果树有一棵为空，只有两个节点都为空才会为True，否则为False
    • 递归判断节点值是否一样，判断左子树是否一样，判断右子树是否一样

  • 复杂度分析

    • 时间复杂度：O(min(n,m))，其中 n 为 p 的节点个数，m 为 q 的节点个数。
      空间复杂度：O(min(n,m))。最坏情况下，二叉树退化成一条链，递归需要 O(min(n,m)) 的栈空间。

对称二叉树

  • 【101.对称二叉树】：https://leetcode.cn/problems/symmetric-tree/description/?envType=study-plan-v2&envId=top-100-liked
  • 题目：给你一个二叉树的根节点 root ， 检查它是否轴对称。
  • 思路：
    • 在相同的树基础上改动
    • 把这棵树拆成左右两棵树，看根节点是否相同
    • 递归看左边的左子树和右边的右子树
    • 递归看左边的右子树和右边的左子树
    • 对称：实际上看一边的左子树的值是否等于另一边对称方向子树的值

平衡二叉树

  • 【110. 平衡二叉树】：https://leetcode.cn/problems/balanced-binary-tree/
  • 题目：给定一个二叉树，判断它是否是 平衡二叉树
    • 平衡二叉树：左右子树高度差的绝对值小于1
  • 思路：基于 二叉树的最大深度
    • 核心在于如果不平衡则递归返回-1
    • 本质上-1是由abs(left_height - right_height) > 1条件下产生的，其他判断-1的过程相当于为这个结果开‘绿灯’，因为递归总是要回到根节点函数的调用的。
  • 复杂度分析
    • 时间复杂度：O(n)，其中 n 为二叉树的节点个数。
    • 空间复杂度：O(n)。最坏情况下，二叉树退化成一条链，递归需要 O(n) 的栈空间。

二叉树的右视图

  • 【199.二叉树的右视图】：https://leetcode.cn/problems/binary-tree-right-side-view/?envType=study-plan-v2&envId=top-100-liked

  • 题目：给定一个二叉树的 根节点 root，想象自己站在它的右侧，按照从顶部到底部的顺序，返回从右侧所能看到的节点值。

  • 思路：

    • 在递归的同时记录节点个数或者递归深度
    • 因为是右视图，先递归右子树，再递归左子树，当某个深度首次到达时，对应的节点就在右视图中。
  • 复杂度分析
    • 时间复杂度：O(n)，其中 n 是二叉树的节点个数。
    • 空间复杂度：O(h)，其中 h 是二叉树的高度。递归需要 O(h) 的栈空间。最坏情况下，二叉树退化成一条链，递归需要 O(n) 的栈空间。

验证二叉搜索树

  • 二叉搜索树：该树所有的左子树的值都小于其根节点，所有的右子树的值都大于其根节点（注意是所有的根不只是最顶层的根）

  • 三种做法

    • 前序遍历：先判断，再递归
    • 中序遍历：大于上一个节点
    • 后序遍历：先递归，再判断

  • 前序遍历

    • 遍历到左子树，更新范围的右边界（左子树<根）
    • 遍历到右子树，更新范围的左边界（右子树>根）
  • 中序遍历
    • 用上一个值与当前值进行比较，需大于上一个节点值
  • 后序遍历
    • 返回的是整个子树的取值范围

  • 前序遍历在某些数据下不需要递归到叶子节点就能返回（比如根节点左儿子的值大于根节点的值，左儿子就不会继续往下递归了），而中序遍历和后序遍历至少要递归到一个叶子节点。从这个角度上来说，前序遍历是最快的。

  • 中序遍历很好地利用了二叉搜索树的性质，使用到的变量最少。

  • 后序遍历的思想是最通用的，即自底向上计算子问题的过程。想要学好动态规划的话，请务必掌握自底向上的思想。

  • 复杂度分析

    • 时间复杂度：O(n)，其中 n 为二叉搜索树的节点个数。

    • 空间复杂度：O(n)。最坏情况下，二叉搜索树退化成一条链（注意题目没有保证它是平衡树），因此递归需要 O(n) 的栈空间。

翻转二叉树

  • 【226.翻转二叉树】：https://leetcode.cn/problems/invert-binary-tree/description/?envType=study-plan-v2&envId=top-100-liked
  • 题目：给你一棵二叉树的根节点 root ，翻转这棵二叉树，并返回其根节点。
  • 核心思路：
    • 对于根节点，它的左右儿子必须交换，即左儿子变成右儿子，右儿子变成左儿子。
    • 对于根节点的左右子树，也需要翻转其内部节点。这是一个和原问题相似的子问题，这可以用递归解决。
  • 思路：
    • 递归调用 invertTree(root.left)，获取到左子树翻转后的结果 left。
    • 递归调用 invertTree(root.right)，获取到右子树翻转后的结果 right。
    • 交换左右儿子，即更新 root.left 为 right，更新 root.right 为 left。
    • 返回 root。
    • 递归边界：如果 root 是空节点，返回空。
  • 也可以先交换左右儿子，然后递归处理左子树，递归处理右子树。
  • 复杂度分析
    • 时间复杂度：O(n)，其中 n 为二叉树的节点个数。
    • 空间复杂度：O(n)。最坏情况下，二叉树退化成一条链，递归需要 O(n) 的栈空间。

二叉树的最近公共祖先

  • 【236. 二叉树的最近公共祖先】：https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/description/?envType=study-plan-v2&envId=top-100-liked
  • 题目：给定一个二叉树, 找到该树中两个指定节点的最近公共祖先
    • 最近公共祖先的定义为：“对于有根树 T 的两个节点 p、q，最近公共祖先表示为一个节点 x，满足 x 是 p、q 的祖先且 x 的深度尽可能大（一个节点也可以是它自己的祖先）。”

  • 难点
    • 如果在递归中找到了p或q，还需要继续递归寻找吗
      • 不需要了。比如，找到了p，无论q是p后续的哪个叶子节点，最近公共祖先都是p
      • 即，如果下面有 q 或者 p，那么当前节点就是最近公共祖先，直接返回当前节点。如果下面没有 q 和 p，那既然都没有要找的节点了，也不需要递归，直接返回当前节点。
    • 如果左右子树都有，如p在左子树，q在右子树，则最近公共祖先为当前节点
    • 只有左子树找到，则返回左子树的递归结果；同理右子树也是
    • lowestCommonAncestor 函数的返回值是什么意思？
      • 返回值的准确含义是「最近公共祖先的候选项」。对于最外层的递归调用者来说，返回值是最近公共祖先的意思。但是，在递归过程中，返回值不一定是最近公共祖先，可能是空节点（表示子树内没找到任何有用信息）、节点 p 或者节点 q（可能成为最近公共祖先，或者用来辅助判断某个节点是否为最近公共祖先）。
  • 复杂度分析
    • 时间复杂度：O(n)，其中 n 为二叉树的节点个数。
    • 空间复杂度：O(n)。最坏情况下，二叉树是一条链，因此递归需要 O(n) 的栈空间。

二叉搜索树的最近公共祖先

  • 【235. 二叉搜索树的最近公共祖先】：https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-search-tree/description/

  • 题目：给定一个二叉搜索树, 找到该树中两个指定节点的最近公共祖先。

    • 最近公共祖先的定义为：“对于有根树 T 的两个结点 p、q，最近公共祖先表示为一个结点 x，满足 x 是 p、q 的祖先且 x 的深度尽可能大（一个节点也可以是它自己的祖先）。”

  • 思路

    • 为什么本题无需判断节点是空节点？
      • 在二叉搜索树的情况下，只有发现 p（或者 q）在左/右子树中才会递归左/右子树，这间接说明递归到的节点一定不是空节点（注意题目保证 p 和 q 一定在树中）。236 题不是二叉搜索树，虽然题目保证 p 和 q 一定在树中，但却无法保证 p（或者 q）一定在我们目前递归到的子树中（因为无法像本题提前判断），所以 236 题无法保证当前节点一定非空。

  • 复杂度分析

    • 时间复杂度：O(n)，其中 n 为二叉搜索树的节点个数。

    • 空间复杂度：O(n)。最坏情况下，二叉搜索树退化成一条链（注意题目没有保证它是平衡树），因此递归需要 O(n) 的栈空间。

二叉树的层序遍历

  • 【102. 二叉树的层序遍历】：https://leetcode.cn/problems/binary-tree-level-order-traversal/?envType=study-plan-v2&envId=top-100-liked
  • 题目：给你二叉树的根节点 root ，返回其节点值的 层序遍历 。 （即逐层地，从左到右访问所有节点）。
  • 思路：双数组或者队列的方法

  • 复杂度分析

    • 时间复杂度：O(n)，其中 n 为二叉树的节点个数。

    • 空间复杂度：O(n)。满二叉树（每一层都填满）最后一层有大约 n/2 个节点，因此队列中最多有 O(n) 个元素，所以空间复杂度是 O(n) 的。

二叉树的锯齿行层序遍历

  • 【103. 二叉树的锯齿形层序遍历】：https://leetcode.cn/problems/binary-tree-zigzag-level-order-traversal/
  • 题目：给你二叉树的根节点 root ，返回其节点值的 锯齿形层序遍历 。（即先从左往右，再从右往左进行下一层遍历，以此类推，层与层之间交替进行）。
  • 思路：基于二叉树层序遍历，用bool型来判断是否需要反转，用python的切片操作实现反转

  • 复杂度分析

    • 时间复杂度：O(n)，其中 n 为二叉树的节点个数。

    • 空间复杂度：O(n)。满二叉树（每一层都填满）最后一层有大约 n/2 个节点，因此队列中最多有 O(n) 个元素，所以空间复杂度是 O(n) 的。

找树左下角的值

  • 【513. 找树左下角的值】：https://leetcode.cn/problems/find-bottom-left-tree-value/description/

  • 题目：给定一个二叉树的 根节点 root，请找出该二叉树的 最底层 最左边 节点的值。假设二叉树中至少有一个节点。

  • 思路：BFS 这棵二叉树，先把右儿子入队，再把左儿子入队，这样最后一个出队的节点就是左下角的节点了。

  • 复杂度分析

    • 时间复杂度：O(n)，其中 n 为二叉树的节点个数。

    • 空间复杂度：O(n)。满二叉树（每一层都填满）最后一层有大约 n/2 个节点，因此队列中最多有 O(n) 个元素，所以空间复杂度是 O(n) 的。
